# CNT 長さ分布データ

CVF 実験で使った CNT 分散液の AFM 観察から抽出した長さ分布データ。Step 3 以降の
モンテカルロシミュレーションで empirical CDF からサンプリングするのに使う。

> **注意**：これらのデータは暫定版で、解析パラメータの見直し等で**更新される可能性
> がある**。コード側ではファイル名にバージョン情報がない前提で読み込む。差し替え
> 時は同じファイル名で上書きする運用とする。

### サンプル対応の注意点

- ラマン測定と AFM 測定は**同じバイアルから取ったサンプル**で行われている前提。
  異なる場合は系統誤差が入るので、対応関係を疑う必要がある。
- `30min_sup` は上澄み採取しているため、長尺バイアスがかかっている可能性がある。
  他 2 サンプルとは取り扱いが異なる。
- 同一実験条件でも観察位置（pos01, pos03 など）によって分布が変わりうる。

## CSV の列の意味

| 列 | 意味 |
|---|---|
| `cc_id` | 連結成分（connected component）に自動で振った識別番号 |
| `length_nm` | 確定した長さ（nm）。集計に使われる値 |
| `adopt_type` | 長さ決定の方法。`cc_longest` = CC 内の最長経路、`polyline` = 経路探索が使えず単純な折れ線で測定 |
| `turn_angle` | 経路中の最小折れ曲がり角度（°）。120° 以上が採用条件。`polyline` では空欄 |
| `e2e_ratio` | 経路長 ÷ 端点間直線距離。1.40 以下が採用条件。1.0 に近いほど真っ直ぐ |
| `h_mean_nm` | 経路上の平均高さ（nm）。バンドルの太さの目安 |
| `h_max_on_path` | 経路上の最大高さ。**10 nm を超えると aggregate の疑い** |
| `end_tangent_angle` | 端点での接線角度。L 字 artifact 検出に使っていたが現在は参考値のみ |
| `n_pixel` | スケルトンの画素数 |
| `endpoints` | 端点の座標（画像上のピクセル位置） |

---

## 採用基準

すべてのレコードは以下を満たす：

- `length_nm >= 100 nm`（ポア径 100 nm。これ以下は脱落するため除外）
- `turn_angle >= 120°`（折れ曲がりすぎを除外、`cc_longest` のみ）
- `e2e_ratio <= 1.40`（曲がりすぎを除外、`cc_longest` のみ）

## 既知の癖・注意点

### 1. `length_nm == 102.4` の張り付き

3 サンプル全てで `length_nm` が 102.4 ちょうどに 5% 程度張り付いている。
これは 100 nm カットオフ近傍のスケルトン解析の人工ピーク。実際には 100 nm 未満
だが切れずに残ったもの。

| サンプル | 102.4 nm 張り付き |
|---|---|
| 8 min | 33 本（5.6%） |
| 15 min | 15 本（3.6%） |
| 30 min sup | 23 本（5.1%） |

シミュレーション上の扱い：empirical CDF サンプリング時は素直にそのまま使う。
ただし「短尺寄りに人工ピークがあること」を結果解釈時に念頭に置く。

### 2. `adopt_type` の分布バイアス

`polyline` は短尺 CNT に偏る（経路探索が成立しないため）。`cc_longest` は
長尺・きれいなものに偏る。両者を混ぜて empirical CDF を作っている。

| サンプル | polyline | cc_longest |
|---|---|---|
| 8 min | 375（64%） | 212（36%） |
| 15 min | 361（87%） | 53（13%） |
| 30 min sup | 337（75%） | 112（25%） |

### 3. ホモジ時間と長さ分布の関係は単調ではない

直感的には「ホモジが長いほど切れて短くなる」が予想されるが、3 サンプルを見ると：

| サンプル | median (nm) | log-normal μ | log-normal σ | L≥334 nm 本数比 |
|---|---|---|---|---|
| 8 min | 177.2 | 5.37 | 0.61 | 19.4% |
| 15 min | **235.8** | **5.60** | 0.70 | **32.9%** |
| 30 min sup | 188.6 | 5.47 | 0.68 | 26.5% |

**15 min が一番長尺リッチ**。考えられる原因：

- サンプルバイアス（観察位置・上澄み採取の有無など）
- ホモジ時間の非単調性（バンドル分散と切断のバランス）
- 8 min サンプルの `polyline` 比率が他より低く、検出特性が異なる

→ 「ホモジ時間 = 切断量」と単純に紐付けるのは危険。各サンプルの分布をそのまま
使い、配向度との対応を一つずつ取るのが安全。

### 4. aggregate チェック

`cc_longest` で記録されている `h_max_on_path` を見る限り、3 サンプル全てで
`h_max < 10 nm`。aggregate（h > 10）は 0 件。データ品質は良好。

### 5. 8 min サンプルについて

実験者によると 8 min サンプルは「ゴミが多い」とのコメントあり。データ上の指標
（h_max など）には現れていないが、解釈時には他 2 サンプル（特に 15 min）の方を
信頼するのが安全かもしれない。

## シミュレーションでの使い方（想定）

```python
import pandas as pd

# empirical サンプリング
df = pd.read_csv("data/length_distributions/15min_pos01_final_lengths.csv")
L_samples = df["length_nm"].sample(n=10000, replace=True, random_state=42).values
# nm → m に変換してから物理計算へ
L_samples_m = L_samples * 1e-9
```

- log-normal フィットを通さず empirical CDF から直接サンプリングする方針。
  フィットだと裾を過小評価する可能性がある。
- 100 nm カットオフはデータ取得時に既にかかっているのでシミュ側で再カットしない。
- `h_max_on_path > 10 nm` のフィルタは現状不要（該当なし）。将来データ更新で
  必要になれば追加する。
