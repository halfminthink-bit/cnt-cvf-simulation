# CNT-CVF Simulation - Claude Code 開発ガイド

## プロジェクト概要

真空ろ過法（CVF: Centrifugal Vacuum Filtration）によるCNT配向プロセスを流体力学的にモデル化するシミュレーション。実験条件（drop間隔・膜構造・CNT長さ分布）と配向度（ラマンGピーク比）の関係を再現することが目的。

---

## 開発の進め方（最重要）

**段階的に実装する。各Stepの終わりで必ず止まって、ユーザーに出力を確認してもらってから次に進む。**

| Step | 内容 | 完了条件 |
|---|---|---|
| 1 | Kawai式による配向度計算のみ | W∈[200,500]nm × L∈[50,1000]nm のテーブル + 配向度 vs L/W のグラフ出力 |
| 2 | 1本のCNTのLangevin軌跡 | 1本のCNTを膜面まで落として、軌跡を可視化 |
| 3 | モンテカルロN本で配向度算出 | 1実験条件で配向度を予測、実験値と比較 |
| 4 | パラメータスイープ | 実験条件全体を再現できるか検証 |
| 5 | モデル拡張（Phase 2） | Step 4で合わない現象だけ堆積層効果等を追加 |

**Step 1〜2を一気に実装しない。Step 1が終わったら必ず止まる。**

### 主要な物理量と式

#### 並進拡散係数
```
D_trans = kT × ln(L/d) / (2πηL)
        ≈ 1.6 × 10⁻¹¹ m²/s
```

#### 回転拡散係数
```
D_rot = 3kT × ln(L/d) / (πηL³)
τ_rot = 1 / (2 D_rot) ≈ 0.3 ms
```

#### ペクレ数
```
Pe = U × L / D_trans
```
実験条件はすべて Pe >> 1（対流支配）。Pe = 1 になる流速は約 70 µm/s。

#### Stokes抗力（ロッド、流れに垂直）
```
F = 4πηLU / [ln(L/d) + 0.5]
F ≈ 数 fN @ U = 数 µm/s
```

#### Kawai式（配向度計算）

溝内のCNT角度分布を一様分布と仮定（θ ∈ [-θ_max, θ_max]）：

- L > W: θ_max = arcsin(W / L)
- L ≤ W: 自由回転（配向なし）→ Gピーク比 = 1.0

Gピーク比:
```
I_∥ / I_⊥ = ∫[-θmax, θmax] (10cos²θ + sin²θ) dθ
           / ∫[-θmax, θmax] (cos²θ + 10sin²θ) dθ
```

係数 10 はラマンの偏光依存項（軸方向 vs 直交方向のG band強度比）。

## コーディング規約

### 単位系
- **すべて SI 単位（m, s, kg, K, Pa, N）で内部計算する**
- nm, µm, fN, ms などの単位は表示時のみ変換
- 変数名に単位サフィックスを付けない（`length` は m 単位、と決め打ち）。ただし docstring には必ず単位を明記

### 型ヒント
- 全関数に型ヒントを付ける
- numpy配列は `np.ndarray` で型ヒント、shape は docstring で明記

### Docstring
- 各関数・クラスに docstring 必須
- パラメータ・戻り値の **物理的意味と単位** を明記
- 数式が出てくる場合は LaTeX 記法で docstring に含める

例:
```python
def translational_diffusion(length: float, diameter: float,
                            viscosity: float, temperature: float) -> float:
    """並進拡散係数を計算する（剛体ロッド近似）.

    D_trans = kT * ln(L/d) / (2π η L)

    Args:
        length: CNT長さ L [m]
        diameter: CNT直径 d [m]
        viscosity: 溶媒粘度 η [Pa·s]
        temperature: 温度 T [K]

    Returns:
        並進拡散係数 [m²/s]
    """
```

### 関数設計
- 1関数1物理プロセス
- 純粋関数を優先（副作用を避ける）
- 物理パラメータはハードコードせず、関数引数または config から読む

### ファイル構成
```
src/cnt_cvf/
├── __init__.py
├── config.py        # 物理定数・固定パラメータ
├── orientation.py   # Kawai式（Step 1）
├── cnt.py           # CNT物性・拡散係数（Step 2以降）
├── membrane.py      # 膜パラメータ・ゾーン定義・流場（Step 2以降）
├── dynamics.py      # Langevin方程式（Step 2以降）
├── capture.py       # 捕捉判定（Step 3以降）
└── runner.py        # モンテカルロ実行（Step 3以降）
```

Step 1で必要なのは `config.py` と `orientation.py` だけ。他は作らない。

## やってはいけないこと

1. **物理パラメータを勝手に「より現実的な値」に書き換えない**
   - 例: η を「DOCだから水より粘度が高いはず」と勝手に変更しない
   - 値を変更したい場合は必ずユーザーに確認
2. **モデルを勝手に「改善」しない**
   - Step 1 で求められているのは Kawai式だけ。「ついでに Langevin も」とやらない
3. **ユーザーが書いた物理コメントや数式を消さない**
4. **単位を混在させない**
   - すべて SI で統一。nm, µm を内部計算で使わない
5. **エラーハンドリングで物理的に不正な値を黙って通さない**
   - 例: L<0, W<0, U<0, T<0 などは raise する
6. **テストを省略しない**
   - 各モジュールに物理的下限値のユニットテストを書く
7. **「既存実装と矛盾するからこっちを変えました」を勝手にやらない**
   - 矛盾を見つけたら報告して、どちらが正しいかユーザーに確認する

## テスト方針

### 物理的に既知の極限でユニットテスト

#### Step 1（Kawai式）
- L < W → 配向度 = 1.0（厳密）
- L = W → 配向度 = 1.0（境界）
- L >> W → θ_max → 0、配向度 → 10（係数の上限）
- 対称性: I_∥/I_⊥ ≥ 1（θ_max ≤ π/2 のとき）

#### Step 2 以降
- D_trans, D_rot が計画書の値の範囲内か（オーダーチェック）
- Stokes抗力が fN オーダーか
- Pe → 0 → 角度分布が等方的（Step 3以降）

## 出力仕様

### Step 1 の出力（必須）
- W ∈ [200, 500] nm、L ∈ [50, 1000] nm のグリッドで配向度を計算
- **テーブル（CSV）**: `results/step1_kawai_table.csv`
- **グラフ（PNG）**: `results/step1_kawai_orientation_vs_LW.png`
  - 横軸: L/W 比（対数スケール推奨）
  - 縦軸: 配向度（I_∥/I_⊥）
  - 複数の W 値（300, 334, 400 nm 等）について曲線を重ねる
  - L/W = 1 の境界線、配向度 = 1, 10 の参照線を入れる
  - 実験条件（W=334nm, L=230nm）の点をマーカーで重ねる

### Step 3 以降の1ラン出力
- ディレクトリ: `results/run_YYYYMMDD_HHMMSS/`
- ファイル:
  - `params.json` - 入力パラメータ全部（乱数シード含む）
  - `summary.json` - 配向度・分類カウント
  - `angles.npy` - 溝捕捉CNTの角度分布（生データ）
  - `trajectories.npz` - 代表軌跡（debug モードのみ、50本程度）

### スイープ出力
- ディレクトリ: `results/sweeps/sweep_<param>_YYYYMMDD/`
- 各条件のサブディレクトリ + 全体の `summary.csv`

## 既知の未解決問題（シミュレーションで解明したい）

1. **配向度3.4が出た理由**: 短尺スタート材料での突出した値。長さ分布の均一性 × 流速の共鳴か？
2. **最適drop間隔の存在理由**: 速すぎても遅すぎても配向度が崩壊する機構
3. **真空2.5で配向度1.1になった理由**: 捕捉はされているのに配向しない
